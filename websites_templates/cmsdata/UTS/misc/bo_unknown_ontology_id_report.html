[tg]Explanation:[/tg]

  • [tg]Categories[/tg] - Ontology categories that client set to be mapped for the given table.
  • [tg]Table[/tg] - OTYS database table name.
  • [tg]Trigger[/tg] - cron - if the ontology service was called from the cron script, manual - if via 'Map Ontology' button in backoffice, insert - if the service was called on inserting new term, update - if the service was called on updating existing term
  • [tg]Suggested Category[/tg] - sometimes Ontology ID cannot be found if the client sets too limiting set of ontology categories for the match criteria table. This column indicated in which ontology category the term would be found if it is present in Ontology database.
